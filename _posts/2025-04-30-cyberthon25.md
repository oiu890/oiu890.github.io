## Treasure manifest
This challenge is an out of bounds read to get the rand seed, followed by reversing the random shuffle to find the flag.  
[download binary](/assets/downloads/cyberthon25/treasure/treasure_manifest) [libc](/assets/downloads/cyberthon25/treasure/libc.so.6)

### Solve
We can try running the program first.  
```bash
QUERY CHEST NUMBER (0 - 9999): 0 
QUERYING CHEST #0.....

Contents: Sacred Salad Spinner
....
....
QUERY CHEST NUMBER (0 - 9999): 10000
QUERYING CHEST #10000.....

Contents: Di

Press Enter to continue...
```
We are given 2 chances to query a chest. The menu tells us to choose a number between 0 and 9999, but if we try and query something outside of that range, nothing stops us. This gives as our out of bound read.  
Throwing the binary into ghidra  
```c
void fill_chest_read_flag(void)

{
  long lVar1;
  time_t cur_time;
  long in_FS_OFFSET;
  int i;
  int j;
  
  lVar1 = *(long *)(in_FS_OFFSET + 0x28);
  time_chunk = (uint *)malloc(4);
  cur_time = time((time_t *)0x0);
  *time_chunk = (uint)cur_time;
  srand(*time_chunk);
  for (i = 0; i < 10000; i = i + 1) {
    *(undefined **)(&chest_arr + (long)i * 8) = PTR_s_Empty_chest..._nothing_but_cobwe_00106020;
  }
  _flag = read_flag();
  for (j = 0; j < 9999; j = j + 1) {
    *(undefined **)(&chest_arr + (long)j * 8) =
         (&PTR_s_Golden_Rubber_Duck_00106040)
         [j + ((j / 6 + (j >> 0x1f) >> 4) - (j >> 0x1f)) * -0x60];
  }
  swap_random_pos(&chest_arr,10000);
  if (lVar1 != *(long *)(in_FS_OFFSET + 0x28)) {
                    /* WARNING: Subroutine does not return */
    __stack_chk_fail();
  }
  return;
}
```
The program mallocs 4 bytes to hold the current time, which is then used as the seed for the random.  
As for the chest array, it first fills arr[0....9999] with a pointer to the string `"Empty chest... nothing but cobwebs"`  
Subsequently, it reads the flag into the _flag variable and fills arr[0....9998] with pointers to different strings.  
```c
void swap_random_pos(long chest_arr,int n)

{
  long lVar1;
  int random_num;
  long in_FS_OFFSET;
  undefined4 i;
  undefined8 cur_item;
  
  lVar1 = *(long *)(in_FS_OFFSET + 0x28);
  for (i = n + -1; 0 < i; i = i + -1) {
    random_num = rand();
    random_num = random_num % (i + 1);
    cur_item = *(undefined8 *)(chest_arr + (long)i * 8);
    *(undefined8 *)(chest_arr + (long)i * 8) = *(undefined8 *)(chest_arr + (long)random_num * 8);
    *(undefined8 *)((long)random_num * 8 + chest_arr) = cur_item;
  }
  if (lVar1 != *(long *)(in_FS_OFFSET + 0x28)) {
                    /* WARNING: Subroutine does not return */
    __stack_chk_fail();
  }
  return;
}
```
swap_random_pos is then called, relying on `rand()` and a shuffling algorithm to mix up the pointers in the array.  
Looking at the ghidra listing, we can see something interesting.     
![flagchest](/assets/images/cyberthon25/flagchest.png)
The chest array, flag and time chunk are all side by side!  
This means that initially, the flag is situated at arr[9999], and time chunk is at arr[10000]. If we can use our out of bounds read to leak the time, we can recreate the shuffling and figure out which index the flag will end up at!!

The flag isn't the last value in the array here cus it already got shuffled. Other than that, everything matches up with what we say in ghidra.  
So anyways, we can just copy the shuffle alg and write a C script to find our flag index.  
```c
//compiling command
//gcc test.c -o test
#include <stdio.h>
#include <stdlib.h>

int main(){
    unsigned int seed;
    scanf("%u",&seed);

    srand(seed);
    int n = 10000;
    int rand_num;
    int arr[10000];
    for(int i=0;i<n;i++){
        arr[i] = i;
    }
    for(int i=n-1;i>0;i--){
        rand_num = rand();
        rand_num = rand_num%(i+1);
        int cur_item = arr[i];
        arr[i] = arr[rand_num];
        arr[rand_num] = cur_item;
    }
    for(int i=0;i<n;i++){
        if(arr[i] == 9999){
            printf("%d",i);
        }
    }
}
```
